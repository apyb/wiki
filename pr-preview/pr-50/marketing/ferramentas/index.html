Licença e uso#

Pela natureza da comunidade, sempre preconizamos o uso de ferramentas open-source: entretanto, para postagem em lotes estamos usando a versão free do Buffer. Consulte a Diretoria Financeira para saber a viabilidade financeira da mensalidade/anuidade/assinatura.

Ferramentas#

Exceto o Canva e o acesso à pasta da APyB, todos os acessos são via email de comunicacao@python.org.br

  • Google Documentos (pasta da APyB)

  • Bit.ly (encurtador de links)

  • HackMD (para estruturação de roteiros em Markdown)

  • Buffer como gerenciador de redes sociais

    • Atualmente é usado para replicar posts do Instagram e LinkedIn

  • Canva (acesso com o email webmaster) para produção das peças gráficas

  • Telegram para disseminação do conteúdo nas redes sociais

  • Linktree como agregador de links (https://wwww.linktr.ee/apyb)

  • Inkscape (ou ferramenta correlata da sua preferência) para edições e ajustes adicionais

Infraestrutura de Comunicação#

  • Verificar com a Diretoria de Tecnologia os acessos ao Canva dos respectivos emails no 1Password

  • Organizar, divulgar e repassar para a Diretoria o formulário de candidatura de cidade-sede

  • Organizar, divulgar e repassar para a Diretoria o formulário de indicações ao Prêmio Dorneles Trémea/Jean Ferri

  • Encomendar a confecção das placas de premiação em aço inox (18x13 cm)

  • Organizar entrega do prêmio em mãos, videochamada gravada ou ao vivo - deve-se realizar tal atividade junto com a Diretoria de Tecnologia e equipe de Infraestrutura da Python Brasil

  • Acionar Presidência e Diretoria Financeira para validar os eventos apoiados, receitas e caixas para a Associação

Workflow#

  • Github Discussions é o canal central de solicitação e de comunicação com a Diretoria

    • O texto das redações é escrito em Markdown

    • Solicitar revisão - caso necessário para a Diretoria

    • Há uma action no CI/CD do apyb/comunidade que dispara email automaticamente para a lista de email dos Associados

    • A imagem do post deve ser única e conter as informações chave de forma clara e sucinta

    • A redação para a issue será o texto base para as postagens do LinkedIn e do Instagram

  • Bluesky/Mastodon/X

    • O texto base anterior é reduzido com foco pro link, data e hora da postagem (caso aplicável)

    • Usar a imagem de banner da issue do Github Discussions

  • LinkedIn e Instagram

    • Usar o carrousel com a mesma arte base do banner para o Github Discussions

    • Realizar os ajustes necessários de marcação dos usernames nas respectivas redes

    • Usar hashtags

    • Agendamento via Buffer ou postagem manual

  • Instagram

    • Reels, Stories e DMs são melhor realizadas via aplicativo móvel - verificar o seu consumo e interação com as redes sociais para melhor fazer uso da rede sem comprometer o seu tempo de tela

  • Telegram

    • É usado o mesmo texto na versão reduzida para o Bluesky/Mastodon e Twitter/X

    • A imagem também é a mesma dessas redes sociais - que é a mesma da issue correspondente no Github

    • Estruturar a mensagem para o seu inbox do Telegram e encaminhar essa mensagem para os grupos - atentar-se para o fato de alguns deles não aceitarem imagens
